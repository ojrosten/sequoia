      10              : /** @file
      11              :     @brief Abstractions pertaining to vector spaces, affine spaces and their generalizations.
      12              : 
      13              :     Representing abstract algebraic structures in C++ presents an interesting
      14              :     challenge. At root, the fundamental abstraction is a set; indeed, a vector
      15              :     space is nothing but a set with some additional structure defined. However,
      16              :     sets of objects are not straightforward to represent, in general, in C++.
      17              : 
      18              :     Consider the real numbers. In C++ we can give a name to the is set
      19              :     by using the type system.
      20              : 
      21              :     struct R {};
      22              : 
      23              :     But what of the elements of R? Here we run into an immediate difficulty. We
      24              :     would like to associate them with the values of a type. But to truly do so
      25              :     we require a type with an infinite number of values (and uncountably so, in this
      26              :     case). Therefore, when seeking representations, particularly of infinite sets,
      27              :     we are generally reduced to approximations.
      28              : 
      29              :     However, at least in so far as the set underpinning a particular vector space
      30              :     goes, it turns out that, for our purposes, we need go no further than naming it.
      31              :     As we will explain momentarily, this is because when dealing with vector spaces
      32              :     in practice, we are almost always interested in the _coordinates_ of a vector
      33              :     with respect to a particular basis and not the abstract entities comprising
      34              :     the elements of the underlying set.
      35              : 
      36              :     Before discussing coordinate systems, it is helpful to be more precise about
      37              :     the fundamental definition of a vector space. The first four axioms pertain
      38              :     just to the elements of the underlying set and amount to stating that a vector
      39              :     space is an abelian group under addition. Thus, vector addition is associative,
      40              :     commutative, has an identity element and admits an inverse.
      41              : 
      42              :     However, the remaining properties of a vector space require not just the underlying
      43              :     set, V, but also a field, F. Canonical examples of a field include the rationals or
      44              :     the reals: sets admitting the standard arithmetic operations and for which every
      45              :     element has both an additive and multiplicative inverse. As such, we speak of
      46              :     a vector space over a field. A vector space admits a binary operation such that
      47              :     any element of V can be multiplied by any element of F, to give another element
      48              :     within V (or the same element for the multiplicative identity).
      49              : 
      50              :     Every vector space admits at least one basis. For a vector space of dimension d,
      51              :     a basis is a set of d elements which are linearly independent and so span the
      52              :     space. Let the basis elements be denoted b_0, ..., b_{n-1}. Any vector in the space
      53              :     may be written as a linear combination:
      54              : 
      55              :     v = a_0 b_0 + ... + a_{n-1} b_{n-1},
      56              : 
      57              :     where the a_i are valued in the field, F. The set of these values [a_0, ..., a_{n-1}]
      58              :     are none other than the coordinates of v with respect to this particular basis.
      59              :     The [a_0, ..., a_{n-1}] are often informally referred to as a vector. However
      60              :     stricty speaking this is an abuse of terminology and conflates two distinct concepts:
      61              :     an actual vector which is an element of the set which forms the vector space,
      62              :     and a representation of this vector via the coordinates with respect to a
      63              :     particular basis. This distinction can be further reinforced by pointing out that
      64              :     two observers who agree they are talking about the same vector (i.e. set element)
      65              :     will nevertheless disagree on the coordinates if they are using different bases -
      66              :     as they are entirely entitled to do! To further add to the confusion, the coordinates
      67              :     may be referred to as a coordinate vector which is perhaps unfortunate.
      68              : 
      69              :     Regardless, from the perspective of performing actual calculations, the coordinates
      70              :     are key. An crucial point to make is that, when dealing with the coordinates,
      71              :     the underlying elements of the set, V, make no explicit appearance. This is a
      72              :     manifestation of the fact that two vector spaces of the same dimension and over
      73              :     the same field are isomorphic. This is incredibly helpful since, for many practical
      74              :     purposes, we need not represent the underlying set beyond, at most, perhaps giving
      75              :     it a name. For example, consider the vector space formed by functions
      76              :     which map some set into a field: the question of how to represent the elements
      77              :     of this vector space in C++ is completely circumvented.
      78              : 
      79              :     However, that is not to say that subtleties of imperfect representations of
      80              :     mathematical abstractions are entirely avoided; indeed, quite the contrary! The
      81              :     coordinates are valued in a field and so at this stage we must deal with the fact that
      82              :     C++ types such as float and double model the real numbers imperfectly. Nevertheless,
      83              :     the burden has been shifted from attempting to represent things in C++ that may
      84              :     be completely infeasible to things which can be done to reasonable approximation.
      85              : 
      86              :     Vector spaces are just one of the things treated in the code that follows. There
      87              :     are several important generalizations. First, there are affine spaces, which comprise
      88              :     a set, A, together with a vector space, V, whose additive group acts freely and
      89              :     transitively on A. Intuitvely, we can start at any point in A and translate to
      90              :     any other point by adding the appropriate vector. In fact, the relationship is
      91              :     stronger than this: choosing any point in A and adding any vector in V will give
      92              :     a point in A. A nice example of an affine space is Euclidean space. Two observers
      93              :     in this space, Alice and Bob, are entirely entitled to define their location as the
      94              :     origin. Neither is more right than the other since this space has no distinguished origin.
      95              :     Alice and Bob will, in general, disagree about the coordinates of points in the space.
      96              :     However, they will agree on the vector which translates from one point to another (though
      97              :     if they compare vector coordinates, they may have ton contend with using different bases
      98              :     on the vector space!).
      99              : 
     100              :     An affine space is sometimes described as a vector space which has forgotten its origin.
     101              :     Indeed, a vector space is an affine space over itself. This is interesting in terms
     102              :     of representing these concepts in C++. Since a vector space is a special case of
     103              :     an affine space, this suggests that an affine space concept is more fundamental,
     104              :     with the vector space being a refinement. However, a vector space is part of the
     105              :     definition of an affine space (a set and a vector space, satisfying certain conditions)
     106              :     and so it is this that will be reflected by the concepts defined below: the affine
     107              :     space concept depends on the vector_space concent, and not vice-versa.
     108              : 
     109              :     It will be useful for our purposes to generalize affine spaces. Consider taking a
     110              :     convex subset, C, of an affine space. We may translate from any point in C to any
     111              :     other by adding the appropriate vector from V. However, there are elements of V
     112              :     which, when added to a point in C will take us outside of C and into the broader
     113              :     affine space into which it is a part. However, we do not want define Convex spaces
     114              :     via an embedding in a bigger space. There seems to be a solution which fits neatly
     115              :     into a C++ implementation. Define a convex space, C, to comprise:
     116              :     1. The union, S', of a set of points, S, and an exception state, E
     117              :     2. A vector space, V
     118              :     such that:
     119              :     A. The difference of any two points in S is in V
     120              :     B. An element of V, when added to S remains either in S or maps to E
     121              :     Note that, since every point in S is mapped by elements of V into E, and there is
     122              :     no mapping from E back into S, the action of the additive group of V is not bijective,
     123              :     violating one of the axioms of affine spaces.
     124              : 
     125              :     The other interesting generialization is to consider relaxing a vector space's field
     126              :     to a ring. The resulting construction is called a module, which is a generalization
     127              :     of a vector space. Our motivation for this is that the integers form a commutative
     128              :     ring and not a field, since integers do not, in general, have multiplicative inverses
     129              :     valued within the integers. Rather than attempting to deal with modules in full generality,
     130              :     we restrict our attention to what may be the most useful, practical cases in the context
     131              :     of C++: free modules over commutative rings. Free modules are those which admit a
     132              :     basis.
     133              : 
     134              :     In line with the above, we also consider affine spaces over free modules and their
     135              :     convex generalization where the action of the free module is not bijective.
     136              : 
     137              :     The final issue to address is that question of why to bother modelling concepts such
     138              :     as vector spaces in the abstract sense if it is their coordinates which are the things
     139              :     of use from the perspective of practical computation. The point is that, for example,
     140              :     vector spaces and affine spaces admit different operations: whereas elements of a vector
     141              :     space can be added, the same is not true of the elements of the set underpinning an
     142              :     affine space. By introducing concepts for the abstract algebraic constructs, we can treat
     143              :     coordinates on all of these spaces in a common way by using constraints to enable
     144              :     or disable specific operations. Thus, the coordinates class template is templated on,
     145              :     amongst other things, a convex_space. To define such a space just requires introducing
     146              :     a struct exposing a small amount of data (types and values) known at compile time.
     147              :     These data determines whether we intend to model a vector space, a free module over a
     148              :     commutative space, an affine space or whatever. This is sufficient for the coordinates
     149              :     implementation to expose the correct set of operations.
     150              :  */
