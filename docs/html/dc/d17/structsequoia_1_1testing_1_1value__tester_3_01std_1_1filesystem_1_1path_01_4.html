Detailed Description

Checks equivalence of filesystem paths.

For the overloads of test accepting either equivalence or weak_equivalence as the first argument, all file comparisons are performed using string_based_file_comparer. Other overloads allow clients to customize the way in which files are compared.

Files are considered equivalent if they have the same name and the same contents; similarly directories.

Files are considered weakly equivalent if they have the same contents; similarly directories. The names of both are ignored.
